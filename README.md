# 서브온 병원동행 서비스 홈페이지

## 1. 프로젝트 구조

- `index.html`: 홈
- `service.html`: 서비스 소개
- `process.html`: 이용 절차
- `managers.html`: 동행매니저
- `care-report.html`: 케어리포트
- `pricing.html`: 이용 요금
- `customer-center.html`: 고객센터
- `terms.html`: 이용약관
- `privacy.html`: 개인정보처리방침
- `sleep-endoscopy.html`: 수면내시경 보호자 동행 랜딩
- `parents-hospital.html`: 부모님 대학병원 진료 동행 랜딩
- `admission-discharge.html`: 입퇴원 동행 랜딩
- `styles.css`: 공통 디자인 시스템
- `script.js`: 모바일 메뉴, FAQ, 버튼 동작

## 2. 프로젝트 목적

서브온 병원동행 서비스의 B2C 신규고객 유입 확대를 위한 정적 HTML 홈페이지입니다. 1차 초안의 SEO 구조는 유지하면서, 목표 디자인 시안에 맞춰 왼쪽 대형 카피, 가운데 어르신·동행매니저 이미지 영역, 오른쪽 진행현황·케어리포트 UI, 신뢰 지표 바를 한 화면에 배치했습니다.

## 2-1. 2026-07-10 홈페이지 프리미엄 리디자인

목표 디자인 시안(공공·기업 신뢰 섹션, 다크 케어리포트 섹션, 5단계 이용 절차, 3카드 요금, 다크그린 Final CTA 등)에 맞춰 `index.html`을 전면 재구성했습니다. 자세한 변경 목록은 대화 로그를 참고하되, 핵심 사항은 다음과 같습니다.

- **적용 범위**: `index.html`, `styles.css`, `script.js`만 수정했습니다. 다른 6개 페이지(service/process/managers/care-report/pricing/customer-center.html)는 건드리지 않았지만, 공용 클래스(`site-header`, `.faq-item`, `.text-link`, `.container`)를 다듬어서 헤더 여백·폰트·FAQ 아코디언 디자인이 사이트 전체에 자연스럽게 반영됩니다. 반대로 `.site-footer`의 밝은 테마는 `index.html`에만 적용되는 `.hp-footer` 보조 클래스로 범위를 좁혀서, 다른 페이지의 기존 다크 푸터는 그대로 유지됩니다.
- **홈페이지 전용 클래스**: 새로 만든 섹션(히어로, 신뢰 지표, 공공·기업 신뢰, 상황별 안내, 케어리포트, 절차, 요금, FAQ, 채널, Final CTA, 푸터 내부 구조)은 모두 `hp-` 접두사 클래스를 사용해 다른 페이지가 쓰는 기존 클래스(`need-grid`, `faq-list`, `pricing-grid` 등)와 충돌하지 않도록 분리했습니다.
- **아이콘**: 외부 아이콘 라이브러리 없이 `index.html` 상단에 인라인 SVG 스프라이트(`<symbol>` 23종)를 직접 추가하고 `<use>`로 재사용했습니다.
- **"왜 서브온인가" 섹션 삭제**: 목표 시안에 없고 신뢰 배지 바(전문 자격 매니저·1:1 면접·범죄경력조회 등)와 내용이 겹쳐 제거했습니다.
- **헤더 메뉴 문구 조정**: 목표 시안은 `이용 후기`(리뷰) 메뉴를 포함하지만, 실제 후기 콘텐츠나 페이지가 없어 후기를 지어내지 않았습니다. 대신 기존 6개 메뉴(서비스 소개/이용 절차/동행매니저/케어리포트/이용 요금/문의하기)를 유지하고 `고객센터`만 `문의하기`로 라벨을 다듬었습니다. **실제 고객 후기를 받으면 `managers.html` 근처에 후기 섹션·메뉴를 추가하는 후속 작업을 권장합니다.**
- **CSS 캐시 버스팅**: `index.html`의 `<link rel="stylesheet">`에 `?v=20260710b` 쿼리를 추가했습니다. 이후 `styles.css`를 수정하면 이 버전 문자열을 올려야 방문자 브라우저가 최신 스타일을 다시 받습니다.
- **레거시 CSS 정리**: 이전 히어로 구조(`.hero`, `.hero-shell`, `.hero-status`, `.trust-bar`, `.trust-grid`, `.trust-feature-strip` 등, 약 640줄)가 완전히 죽은 코드가 되어 `styles.css`에서 삭제했습니다. 다만 `styles.css`는 과거 여러 차례의 부분 수정이 누적된 5000줄 이상의 파일이라, `.proof-*`·`.need-grid`·`.report-section`·`.pricing-grid`(구버전)처럼 이번에 `hp-*`로 대체되어 더는 쓰이지 않지만 아직 남아 있는 소규모 블록이 있을 수 있습니다. 동작에는 영향이 없으나, 다음 정리 작업 때 함께 제거하는 것을 권장합니다.

### 확인이 필요한 항목

- **요금 기준 반영**: 제공된 병원동행 서비스 신청 화면 기준으로 베이직 55,000원, 시간쿠폰제 220,000원, 내시경 50,000원을 홈 요금 카드, 요금 페이지, FAQ, JSON-LD에 반영했습니다. 신청시간 초과 시 1시간당 20,000원, 15분 단위 5,000원 후불 청구 문구도 함께 통일했습니다.
- **이용약관 / 개인정보처리방침**: 이용약관과 개인정보처리방침 페이지를 연결했습니다. 위치기반서비스 이용약관은 실제 원문과 적용 여부가 확정되지 않아 푸터에서 숨김 처리했습니다.
- **히어로 이미지**: 목표 시안이 제시한 경로(`./assets/images/hero-patient-manager.png`) 대신, 이미 프로젝트에 있던 동일 사진 파일 `assets/serveon-hero-main.jpg`를 그대로 재사용했습니다.
- **아이콘 스타일**: 손으로 그린 미니멀 라인 아이콘 23종을 새로 만들어 사용했습니다. 브랜드 가이드에 맞는 아이콘 세트가 따로 있다면 교체가 필요합니다.

## 3. 실행 방법

프레임워크와 빌드 도구를 사용하지 않는 정적 페이지입니다.

1. `index.html`을 브라우저에서 직접 엽니다.
2. `styles.css`, `script.js`가 같은 폴더에 있어야 합니다.
3. 배포 시 공식 도메인 `https://www.theserveon.com/`에 업로드하고 실제 채널 URL을 최종 반영합니다.

- 보조채널: 구글 블로거는 구글 검색 보조 채널, 티스토리는 다음 검색 보조 채널, 스마트스토어는 상담 후 결제 링크 채널, 인스타그램은 신뢰 형성·재노출 채널, YouTube는 대표 신뢰 영상·브랜드 설명 보강 채널로 정리했습니다.

## 7. 디자인 개선 사항

- desktop 기준 최대 콘텐츠 폭을 1240px로 확장했습니다.
- Hero를 `왼쪽 카피 + 가운데 이미지 영역 + 오른쪽 UI 카드` 3열 구조로 재구성했습니다.
- 실제 사진 교체를 고려해 hero 이미지 `img` 태그를 유지하고, 이미지가 없을 때는 CSS fallback visual이 보이도록 구성했습니다.
- 오른쪽에 `오늘의 동행 진행 현황` timeline과 floating 케어리포트 UI를 구성했습니다.
- Hero 바로 아래 딥네이비 신뢰 지표 바를 추가했습니다.
- `왜 서브온인가` 섹션은 전문 자격, 엄격한 검증, 실시간 안내, 보호자 안심 4개 대형 카드로 정리했습니다.
- 서비스 절차는 번호 배지와 아이콘이 있는 timeline형 카드로 개선했습니다.
- 케어리포트 섹션은 딥그린/네이비 배경과 스마트폰 mockup UI로 차별화했습니다.
- 요금 섹션에는 `포함 서비스` 패널을 추가했습니다.
- 모바일 390px 전후에서 텍스트, 버튼, 카드가 1열로 자연스럽게 쌓이도록 반응형을 정리했습니다.

## 8. SEO 메타태그 설명

- title: `서브온`, `병원동행 전문 서비스`, `부모님 병원동행`, `케어리포트`를 함께 배치해 브랜드와 서비스 의도를 강화했습니다.
- meta description: 부모님 병원동행, 수면내시경 보호자 동행, 항암·재활·입퇴원 동행, 접수부터 귀가까지의 지원 범위, 상담 전화번호를 포함했습니다.
- canonical: 공식 도메인 `https://www.theserveon.com/`으로 설정했습니다.
- Open Graph: 공유 시 `서브온 병원동행 서비스`와 케어리포트 중심 메시지가 노출되도록 설정했습니다.

각 HTML 페이지는 고유한 title, meta description, canonical, Open Graph URL을 갖습니다. 공통 Organization JSON-LD를 각 페이지에 넣고, 홈과 고객센터 등 FAQ가 필요한 페이지에는 FAQPage JSON-LD를 추가했습니다.

## 9. JSON-LD 설명

`index.html`에 `script type="application/ld+json"`으로 구조화 데이터를 포함했습니다.

- Organization: 상호, 법인명, 대체명, 공식 URL, 대표 전화, 이메일, 사업자등록번호, 네이버 블로그, YouTube를 연결합니다.
- LocalBusiness: 병원동행 서비스 회사의 NAP 정보, 이메일, 사업자등록번호, 영업시간을 명시합니다.
- Service: `서브온 병원동행 서비스`의 서비스 유형, 제공자, 제공 지역, 지원 범위를 설명합니다.
- FAQPage: 자주 묻는 질문 10개를 구조화해 검색 결과와 AI 브리핑 대응 가능성을 높였습니다.

- 서비스 소개에는 수면내시경 보호자 동행, 부모님 대학병원 진료, 항암·투석·재활 정기 일정, 원거리 가족 상황을 검색 의도별 상담 포인트로 보강했습니다.
- 이용 절차는 8단계에서 병원 일정 상담, 매니저 프로필 안내, 전일 해피콜·예약 재확인, 결제 안내·만족도 확인을 포함한 10단계 흐름으로 확장했습니다.
- 동행매니저 페이지에는 배정 전 프로필 안내, 전일 확인, 동행·안전·서비스 교육, 이용 후 확인 절차를 보강했습니다.
- 케어리포트 페이지에는 동행 일정, 방문 단계, 안내받은 내용, 처방 약품, 다음 일정, 보호자 전달사항 등 실제 확인 항목을 추가했습니다.
- 요금과 고객센터 페이지에는 예상 이용료 문의, 가능 여부 확인, 카카오톡·전화 1분 문의 중심의 전환 문구를 보강했습니다.
- FAQ가 없던 서비스 소개, 이용 절차, 동행매니저, 케어리포트 페이지에 화면 FAQ와 FAQPage JSON-LD를 추가했습니다.
- 모바일 전환을 위해 전화 상담, 카카오톡 문의, 가능 여부 확인 하단 고정 CTA를 추가했습니다.
- GA4 연결 전에도 `dataLayer`에 기록될 수 있도록 `click_hero_consult`, `click_kakao`, `click_phone`, `click_situation_cta`, `view_pricing`, `open_faq`, `submit_consult_form` 이벤트명을 `script.js`에 반영했습니다.

## 13. 공공·기업 신뢰 자산 반영

`index.html` 신뢰 지표 바와 `왜 서브온인가` 섹션 사이에 `공공기관과 기업이 선택한 병원동행 서비스` 섹션(`.proof-section`)을 추가했습니다. 회사소개서(`서브온 병원동행서비스 회사소개서__260625.pdf`)를 근거로 다음 세 카드로 구성했습니다.

- **Public Reference (서울시·보건복지부 공공 병원동행 수행 경험)**: 2025년 서울시 병원 안심동행서비스 수행기관 선정, 2026년 서울시 1인가구 동행서비스 수행기관 선정, 서울시 서초구청 돌봄SOS센터 업무협약, 보건복지부 일상돌봄(특화) 병원동행서비스 제공, 서울시 서초복지돌봄재단 서초온(ON) 통합돌봄 서비스 제공 협약, 특허청 산하 한국발명진흥회 병원동행서비스 제공
- **Enterprise Partnership (대기업·보험사·헬스케어 제휴 경험)**: 삼성생명 경인법인지역단, 현대제철·현대 ISC, 메가스터디교육, ABB 코리아·네오위즈 사내복지 도입 계약, 삼성화재·삼성생명·DB손보 보험가입 고객 대상 서비스 영역. 그 외 다수 헬스케어·라이프케어 기업 제휴는 목록을 나열하지 않고 한 줄로 요약했습니다.
- **Certified Operation (사회서비스 기반의 운영 신뢰)**: 보건복지형 예비사회적기업 지정, 사회서비스 제공자 등록, 병원동행매니저 매칭프로그램 저작권 등록, 매니저 자격 요건(사회복지사·요양보호사·간호사·간호조무사 중 1개 이상)

또한 신뢰 지표 바의 마지막 항목을 `공공 병원동행 수행 경험`에서 `서울시 안심동행 · 수행기관 선정`으로 구체화하고, `index.html`의 Organization/LocalBusiness/Service JSON-LD에 `knowsAbout`, `areaServed`, `serviceType`(배열)을 보강했습니다. 기존 Organization·LocalBusiness·Service·FAQPage 구조와 H1·title·meta description·canonical은 그대로 유지했습니다.

**주의 사항**
- 제휴사 로고는 사용하지 않고 텍스트 기반 카드로만 표현했습니다. 로고를 추가하려면 각 기업의 로고 사용 허가 여부를 먼저 확인해야 합니다.
- 회사소개서에 근거가 있는 표현만 사용했고, 과장된 최상급 표현, 순위 표현, 확정 보장으로 오해될 수 있는 표현, 공식 인증으로 오해될 수 있는 표현은 넣지 않았습니다.
- 실제 배포 전 각 협약·계약의 현재 유효 여부, 명칭 표기, 사용 가능 문구는 서브온 대표 확인이 필요합니다.
