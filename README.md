# 서브온 병원동행 서비스 HTML 데모

## 1. 프로젝트 구조

- `index.html`: 홈
- `service.html`: 서비스 소개
- `process.html`: 이용 절차
- `managers.html`: 동행매니저
- `care-report.html`: 케어리포트
- `pricing.html`: 이용 요금
- `customer-center.html`: 고객센터
- `styles.css`: 공통 디자인 시스템
- `script.js`: 모바일 메뉴, FAQ, 버튼 동작

## 2. 프로젝트 목적

서브온 병원동행 서비스의 B2C 신규고객 유입 확대를 위한 대표 보고용 정적 HTML 홈페이지 데모입니다. 1차 초안의 SEO 구조는 유지하면서, 목표 디자인 시안에 맞춰 왼쪽 대형 카피, 가운데 어르신·동행매니저 이미지 영역, 오른쪽 진행현황·케어리포트 UI, 신뢰 지표 바를 한 화면에 배치했습니다.

## 3. 실행 방법

프레임워크와 빌드 도구를 사용하지 않는 정적 페이지입니다.

1. `index.html`을 브라우저에서 직접 엽니다.
2. `styles.css`, `script.js`가 같은 폴더에 있어야 합니다.
3. 배포 시 공식 도메인 `https://www.theserveon.com/`에 업로드하고 실제 채널 URL을 최종 반영합니다.

## 4. 각 페이지 목적

- 홈: 브랜드 첫인상, 핵심 차별화 메시지, 신뢰 지표, 주요 섹션 요약
- 서비스 소개: 서브온 병원동행 서비스의 지원 범위 설명
- 이용 절차: 상담부터 케어리포트 전달까지 단계별 흐름 안내
- 동행매니저: 전문 자격, 검증 절차, 보호자 안내 체계 설명
- 케어리포트: 병원동행 이후 확인 가능한 기록 전달 가치 설명
- 이용 요금: 기본 요금, 포함 서비스, 상담 시 확인 필요 조건 안내
- 고객센터: 전화, 카카오톡, 홈페이지 상담 연결과 상담 전 준비 정보 안내

## 5. 메뉴별 URL

- 서비스 소개: `service.html`
- 이용 절차: `process.html`
- 동행매니저: `managers.html`
- 케어리포트: `care-report.html`
- 이용 요금: `pricing.html`
- 고객센터: `customer-center.html`

## 6. 검색 이슈 반영 사항

- 네이버: title, H1, meta description, 본문, FAQ에 `서브온 병원동행` 신호를 유지했습니다. H1은 `서브온 병원동행 서비스`이며 화면상 핵심 카피는 `동행만 하는 병원동행이 아닙니다. 보호자가 확인할 수 있는 병원동행입니다.`로 강화했습니다.
- 구글: Organization, LocalBusiness, Service JSON-LD를 유지하고, FAQ가 있는 페이지에는 FAQPage JSON-LD를 추가했습니다. footer에는 상호, 서비스명, 주소, 전화번호, 운영시간, 홈페이지 URL을 명확히 배치했습니다.
- 다음·카카오: 공식 채널 안내 섹션에 카카오 상담, 카카오맵/네이버 지도, 티스토리, 스마트스토어, 인스타그램 역할을 연결했습니다. 실제 URL이 확정되지 않은 채널은 `href="#"`로 두고 HTML 주석을 유지했습니다.
- 보조채널: 구글 블로거는 구글 검색 보조 채널, 티스토리는 다음 검색 보조 채널, 스마트스토어는 상담 후 결제 링크 채널, 인스타그램은 신뢰 형성·재노출 채널, YouTube는 대표 신뢰 영상·브랜드 설명 보강 채널로 정리했습니다.

## 7. 디자인 개선 사항

- desktop 기준 최대 콘텐츠 폭을 1240px로 확장했습니다.
- Hero를 `왼쪽 카피 + 가운데 이미지 영역 + 오른쪽 UI 카드` 3열 구조로 재구성했습니다.
- 실제 사진 교체를 고려해 hero 이미지 `img` 태그를 유지하고, 이미지가 없을 때는 CSS fallback visual이 보이도록 구성했습니다.
- 오른쪽에 `오늘의 동행 진행 현황` timeline과 floating 케어리포트 UI를 구성했습니다.
- Hero 바로 아래 딥네이비 신뢰 지표 바를 추가했습니다.
- `왜 서브온인가` 섹션은 전문 자격, 엄격한 검증, 실시간 안내, 보호자 안심 4개 대형 카드로 정리했습니다.
- 서비스 절차는 번호 배지와 아이콘이 있는 timeline형 카드로 개선했습니다.
- 케어리포트 섹션은 딥그린/네이비 배경과 스마트폰 mockup UI로 차별화했습니다.
- 요금 섹션에는 `포함 서비스` 패널을 추가했습니다.
- 모바일 390px 전후에서 텍스트, 버튼, 카드가 1열로 자연스럽게 쌓이도록 반응형을 정리했습니다.

## 8. SEO 메타태그 설명

- title: `서브온`, `병원동행 전문 서비스`, `부모님 병원동행`, `케어리포트`를 함께 배치해 브랜드와 서비스 의도를 강화했습니다.
- meta description: 부모님 병원동행, 수면내시경 보호자 동행, 항암·재활·입퇴원 동행, 접수부터 귀가까지의 지원 범위, 상담 전화번호를 포함했습니다.
- canonical: 공식 도메인 `https://www.theserveon.com/`으로 설정했습니다.
- Open Graph: 공유 시 `서브온 병원동행 서비스`와 케어리포트 중심 메시지가 노출되도록 설정했습니다.

각 HTML 페이지는 고유한 title, meta description, canonical, Open Graph URL을 갖습니다. 공통 Organization JSON-LD를 각 페이지에 넣고, 홈과 고객센터 등 FAQ가 필요한 페이지에는 FAQPage JSON-LD를 추가했습니다.

## 9. JSON-LD 설명

`index.html`에 `script type="application/ld+json"`으로 구조화 데이터를 포함했습니다.

- Organization: 상호, 대체명, 공식 URL, 대표 전화, 네이버 블로그, YouTube를 연결합니다.
- LocalBusiness: 병원동행 서비스 회사의 NAP 정보와 영업시간을 명시합니다.
- Service: `서브온 병원동행 서비스`의 서비스 유형, 제공자, 제공 지역, 지원 범위를 설명합니다.
- FAQPage: 자주 묻는 질문 10개를 구조화해 검색 결과와 AI 브리핑 대응 가능성을 높였습니다.

## 13. 공공·기업 신뢰 자산 반영

`index.html` 신뢰 지표 바와 `왜 서브온인가` 섹션 사이에 `공공기관과 기업이 선택한 병원동행 서비스` 섹션(`.proof-section`)을 추가했습니다. 회사소개서(`서브온 병원동행서비스 회사소개서__260625.pdf`)를 근거로 다음 세 카드로 구성했습니다.

- **Public Reference (서울시·보건복지부 공공 병원동행 수행 경험)**: 2025년 서울시 병원 안심동행서비스 수행기관 선정, 2026년 서울시 1인가구 동행서비스 수행기관 선정, 서울시 서초구청 돌봄SOS센터 업무협약, 보건복지부 일상돌봄(특화) 병원동행서비스 제공, 서울시 서초복지돌봄재단 서초온(ON) 통합돌봄 서비스 제공 협약, 특허청 산하 한국발명진흥회 병원동행서비스 제공
- **Enterprise Partnership (대기업·보험사·헬스케어 제휴 경험)**: 삼성생명 경인법인지역단, 현대제철·현대 ISC, 메가스터디교육, ABB 코리아·네오위즈 사내복지 도입 계약, 삼성화재·삼성생명·DB손보 보험가입 고객 대상 서비스 영역. 그 외 다수 헬스케어·라이프케어 기업 제휴는 목록을 나열하지 않고 한 줄로 요약했습니다.
- **Certified Operation (사회서비스 기반의 운영 신뢰)**: 보건복지형 예비사회적기업 지정, 사회서비스 제공자 등록, 병원동행매니저 매칭프로그램 저작권 등록, 매니저 자격 요건(사회복지사·요양보호사·간호사·간호조무사 중 1개 이상)

또한 신뢰 지표 바의 마지막 항목을 `공공 병원동행 수행 경험`에서 `서울시 안심동행 · 수행기관 선정`으로 구체화하고, `index.html`의 Organization/LocalBusiness/Service JSON-LD에 `knowsAbout`, `areaServed`, `serviceType`(배열)을 보강했습니다. 기존 Organization·LocalBusiness·Service·FAQPage 구조와 H1·title·meta description·canonical은 그대로 유지했습니다.

**주의 사항**
- 제휴사 로고는 사용하지 않고 텍스트 기반 카드로만 표현했습니다. 로고를 추가하려면 각 기업의 로고 사용 허가 여부를 먼저 확인해야 합니다.
- "업무협약", "도입 계약", "수행기관 선정 이력"처럼 회사소개서에 근거가 있는 표현만 사용했고, "최고", "1위", "보증", "인증 파트너"처럼 법적 오해 소지가 있는 표현은 넣지 않았습니다.
- 실제 배포 전 각 협약·계약의 현재 유효 여부, 명칭 표기, 사용 가능 문구는 서브온 대표 확인이 필요합니다.

## 표현 운영 기준

허위 고객 발언, 과장 표현, 의료 행위나 소견 해석처럼 보일 수 있는 문구는 넣지 않았습니다. 케어리포트는 `안내받은 내용 전달`, `기록 확인`, `다음 일정 확인` 중심으로 설명했습니다.
